# PRD: 유튜브 랭킹 & 분석 플랫폼

## TL;DR

**문제 재정의**
- 녹스인플루언서는 멀티 플랫폼(유튜브, 틱톡, 인스타그램, 트위치, 틱톡샵)을 모두 다루는 종합 플랫폼
- 사용자는 **유튜브에만 특화된 깊이 있는 랭킹/분석 서비스**를 원함
- 다른 플랫폼 기능과 제품/서비스 섹션은 제거하고, 유튜브 데이터 분석에 집중

**핵심 가치 제안**
- 유튜브 채널 랭킹 (다양한 정렬 기준)
- 유튜브 카테고리별 심화 분석
- 실시간/주간/월간 트렌드 추적
- 채널 상세 분석 및 비교 기능
- **광고 수익화 최적화**: 빌더 형태의 광고 관리자, 픽셀 광고, SEO 최적화
- **하이브리드 웹 최적화**: 모바일/데스크톱 모두 최적화된 성능

---

## 1. 제품 개요

### 1.1 제품 비전
유튜브 크리에이터, MCN, 마케터를 위한 **유튜브 전용 랭킹 & 분석 플랫폼**

### 1.2 타겟 사용자
1. **유튜브 크리에이터**: 경쟁 채널 분석, 트렌드 파악
2. **MCN/에이전시**: 소속 크리에이터 모니터링, 신규 크리에이터 발굴
3. **마케터/브랜드**: 협업 채널 선정, 인플루언서 마케팅 전략 수립
4. **미디어/연구자**: 유튜브 트렌드 분석, 리포트 작성

### 1.3 핵심 차별화 포인트
- ✅ **유튜브 100% 집중**: 다른 플랫폼 기능 제거로 더 깊이 있는 분석
- ✅ **카테고리별 특화**: 카테고리별 랭킹, 트렌드, 인사이트 제공
- ✅ **실시간 데이터**: 주간/월간 업데이트, 변화 추적
- ✅ **비교 분석**: 채널 간 비교, 성장률 추이 분석
- ✅ **광고 수익화**: 빌더 형태의 광고 관리자로 UI/UX 침해 없는 광고 배치
- ✅ **SEO 최적화**: 검색 엔진 최적화로 유기적 트래픽 확보
- ✅ **하이브리드 웹**: 모바일/데스크톱 모두 최적화된 성능

## 2. 기능 요구사항

### 2.1 메인 랭킹 페이지

#### 2.1.1 랭킹 필터
| 필터 항목 | 옵션 | 설명 |
|---------|------|------|
| **카테고리** | 전체 / 엔터테인먼트 / 음악 / 교육 / 게임 / 스포츠 / 뉴스/정치 / 인물/블로그 / 노하우/스타일 / 기타 | 카테고리별 필터링 |
| **정렬 기준** | 구독자 수 (주간) / 구독자 수 (전체) / 조회수 (주간) / 조회수 (전체) / 성장률 / 참여율 | 정렬 옵션 |
| **기간** | 실시간 / 주간 / 월간 | 데이터 기간 선택 |
| **지역** | 전체 / 한국 / 미국 / 일본 / 기타 | 지역별 필터 (선택) |

#### 2.1.2 랭킹 테이블 컬럼
| 컬럼명 | 설명 | 표시 형식 |
|--------|------|----------|
| **순위** | 현재 순위 (변동 표시: ↑↓) | 숫자 + 화살표 |
| **채널명** | 채널명 + @핸들 | 링크 (상세 페이지 이동) |
| **카테고리** | 채널 카테고리 | 태그 형태 |
| **구독자 수** | 현재 구독자 수 + 주간 변화율 | 예: 4.53억 (+0.2%) |
| **주간 조회수** | 주간 조회수 + 변화율 | 예: 7890.97만 (+23.6%) |
| **총 조회수** | 누적 총 조회수 | 예: 7.89억 |
| **참여율** | 평균 참여율 (좋아요/댓글/공유) | 예: 1.78% |
| **업데이트 날짜** | 데이터 최종 업데이트 일시 | YYYY-MM-DD |

#### 2.1.3 랭킹 테이블 기능
- ✅ 페이지네이션 (20/50/100개씩)
- ✅ 정렬 가능 (컬럼 클릭)
- ✅ 데이터 내보내기 (CSV/Excel)
- ✅ 즐겨찾기/북마크
- ✅ 채널 상세 페이지 이동

### 2.2 채널 상세 페이지

#### 2.2.1 기본 정보

- 구독자 수, 총 조회수, 총 동영상 수
- 카테고리, 채널 생성일
- 채널 설명

- 구독자 수 추이 (주간/월간/연간)
- 조회수 추이 (주간/월간/연간)
- 성장률 그래프
- 예상 수익 추정 (선택)

#### 2.2.3 최근 동영상 분석
- 최근 업로드 동영상 리스트
- 각 동영상별 조회수, 좋아요, 댓글 수
- 평균 조회수, 참여율

#### 2.2.4 인사이트
- 주요 키워드 분석
- 최고 성과 동영상
- 성장 패턴 분석
- 예상 수익 범위

### 2.7 광고 관리 시스템 (빌더 형태)

#### 2.7.1 광고 빌더 에디터
- **드래그 앤 드롭 인터페이스**: 직관적인 광고 배치
- **위치 선택**: 헤더, 사이드바, 콘텐츠 사이, 푸터 등
- **반응형 미리보기**: 모바일/태블릿/데스크톱 미리보기
- **실시간 편집**: 변경사항 즉시 반영

#### 2.7.2 광고 타입
| 광고 타입 | 설명 | 배치 위치 |
|---------|------|----------|
| **배너 광고** | 이미지/HTML 배너 | 헤더, 사이드바, 콘텐츠 사이 |
| **네이티브 광고** | 콘텐츠와 자연스럽게 통합 | 랭킹 테이블 사이, 카드 형태 |
| **픽셀 광고** | 추적 픽셀 (Facebook, Google 등) | 페이지 전체 (보이지 않음) |
| **인피드 광고** | 콘텐츠 흐름에 자연스럽게 삽입 | 랭킹 리스트 중간 |
| **스티키 광고** | 스크롤 시 고정 | 사이드바, 하단 바 |

#### 2.7.3 광고 설정
- **표시 조건**: 페이지별, 카테고리별, 사용자별
- **노출 빈도**: 사용자당 일일 노출 제한
- **스케줄링**: 시작일/종료일 설정
- **A/B 테스트**: 여러 광고 버전 테스트
- **클릭률/노출수 통계**: 실시간 대시보드

#### 2.7.4 픽셀 광고 관리
- **Facebook Pixel**: 전환 추적, 리타겟팅
- **Google Analytics**: 이벤트 추적
- **Google Tag Manager**: 통합 태그 관리
- **커스텀 픽셀**: 직접 픽셀 코드 추가
- **이벤트 추적**: 페이지뷰, 클릭, 전환 등

#### 2.7.5 광고 성과 분석
- **노출수 (Impressions)**: 광고 노출 횟수
- **클릭수 (Clicks)**: 광고 클릭 횟수
- **CTR (Click-Through Rate)**: 클릭률
- **수익 (Revenue)**: 광고 수익
- **eCPM**: 천 노출당 수익
- **시간대별 분석**: 시간대별 성과 분석

#### 2.8.1 메타 태그 관리

#### 2.8.3 사이트맵 & 로봇 설정

#### 2.8.4 콘텐츠 최적화
- **키워드 최적화**: 유튜브 랭킹 관련 키워드
- **내부 링크 구조**: 관련 페이지 연결
- **이미지 최적화**: Alt 태그, WebP 포맷
- **페이지 속도 최적화**: Core Web Vitals 개선

#### 2.8.5 SEO 분석 도구
- **키워드 순위 추적**: 주요 키워드 순위 모니터링
- **백링크 분석**: 외부 링크 추적
- **페이지 속도 분석**: 성능 지표 모니터링
- **SEO 리포트**: 주간/월간 SEO 리포트

## 5. UI/UX 설계

## 6. 기술 스택 제안

### 6.1 프론트엔드
- **Framework**: Next.js 14+ (App Router, Server Components)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Charts**: Recharts / Chart.js
- **State Management**: Zustand / React Query
- **UI Components**: shadcn/ui
- **빌더 라이브러리**: React DnD / Dnd Kit (드래그 앤 드롭)
- **이미지 최적화**: Next.js Image, WebP 포맷
- **성능 최적화**: React Suspense, Streaming SSR

### 6.2 백엔드
- **API**: Next.js API Routes / tRPC (타입 안전성)
- **Database**: PostgreSQL (Prisma ORM)
- **Caching**: Redis (랭킹 데이터 캐싱, 광고 데이터)
- **Data Processing**: Node.js / Python (선택)
- **CDN**: Vercel Edge Network / Cloudflare (정적 자산)
- **Rate Limiting**: API 호출 제한

### 6.3 데이터 수집
- **YouTube Data API v3**: 공식 API 활용
- **스크래핑**: Puppeteer / Playwright (필요시)
- **스케줄링**: Cron Jobs (주간/월간 업데이트)
- **API 최적화**: 배치 처리, 병렬 요청, 캐싱 전략

### 6.4 성능 최적화
- **API 응답 속도**: < 200ms (캐시 히트 시)
- **페이지 로딩**: < 2초 (First Contentful Paint)
- **이미지 최적화**: WebP, 지연 로딩, 사이즈 최적화
- **코드 스플리팅**: 라우트별, 컴포넌트별 분리
- **서버 사이드 렌더링**: SEO 최적화, 초기 로딩 속도
- **Edge Functions**: 지역별 최적화된 응답

### 6.5 SEO 최적화 기술
- **Next.js Metadata API**: 동적 메타 태그 생성
- **구조화된 데이터**: JSON-LD 형식
- **사이트맵**: 동적 생성 (next-sitemap)
- **로봇 설정**: robots.txt 자동 생성
- **Open Graph**: 소셜 미디어 최적화
- **Core Web Vitals**: LCP, FID, CLS 최적화

## 11. API 설계 (상세)

### 11.1 API 아키텍처
- **타입 안전성**: tRPC 사용으로 프론트엔드-백엔드 타입 공유
- **캐싱 전략**: Redis를 통한 다층 캐싱
- **배치 처리**: 여러 채널 데이터를 한 번에 조회
- **Rate Limiting**: API 호출 제한으로 안정성 확보

### 11.2 주요 API 엔드포인트

#### 11.2.1 랭킹 API
```typescript
// 랭킹 조회
GET /api/rankings
Query: {
  category?: string;
  sortBy?: 'subscribers' | 'views' | 'growth' | 'engagement';
  period?: 'realtime' | 'weekly' | 'monthly';
  country?: string;
  page?: number;
  limit?: number;
}
Response: {
  rankings: YouTubeChannel[];
  total: number;
  page: number;
  limit: number;
}

// 채널 상세
GET /api/channels/:id
Response: YouTubeChannel & {
  growthChart: GrowthData[];
  recentVideos: Video[];
  insights: ChannelInsights;
}
```

#### 11.2.2 광고 API
```typescript
// 광고 목록 (관리자)
GET /api/admin/ads
Response: Ad[]

// 광고 생성/수정 (관리자)
POST /api/admin/ads
PUT /api/admin/ads/:id
Body: Ad

// 활성 광고 조회 (프론트엔드)
GET /api/ads/active
Query: {
  page: string;
  location: string;
}
Response: Ad[]

// 광고 성과
GET /api/admin/ads/:id/performance
Query: {
  startDate: string;
  endDate: string;
}
Response: AdPerformance[]
```

#### 11.2.3 픽셀 API
```typescript
// 픽셀 목록 (관리자)
GET /api/admin/pixels
Response: Pixel[]

// 픽셀 생성/수정 (관리자)
POST /api/admin/pixels
PUT /api/admin/pixels/:id

// 활성 픽셀 조회 (프론트엔드)
GET /api/pixels/active
Response: Pixel[]
```

### 11.3 성능 최적화 전략

#### 11.3.1 캐싱 계층
```
1. 브라우저 캐시 (Static Assets)
2. CDN 캐시 (Edge Network)
3. Redis 캐시 (API 응답)
4. 데이터베이스 (원본 데이터)
```

#### 11.3.2 캐싱 전략
- **랭킹 데이터**: 1시간 캐시 (주간 업데이트)
- **채널 상세**: 30분 캐시
- **광고 데이터**: 5분 캐시
- **픽셀 데이터**: 1시간 캐시

#### 11.3.3 배치 처리
- **채널 데이터 수집**: 배치로 여러 채널 동시 조회
- **랭킹 계산**: 백그라운드 작업으로 미리 계산
- **광고 성과 집계**: 일일 배치로 집계

## 12. 광고 수익화 전략

### 12.1 광고 배치 원칙
1. **UI/UX 침해 최소화**: 콘텐츠 흐름을 방해하지 않음
2. **자연스러운 통합**: 네이티브 광고 우선 사용
3. **반응형 최적화**: 디바이스별 최적화된 광고 크기
4. **성능 영향 최소화**: 지연 로딩, 비동기 로드

### 12.2 광고 타입별 배치 전략
- **배너 광고**: 헤더, 사이드바, 푸터 (고정 위치)
- **네이티브 광고**: 콘텐츠 카드 형태로 자연스럽게 통합
- **인피드 광고**: 랭킹 리스트 중간 (5-10개마다 1개)
- **스티키 광고**: 사이드바 고정 (모바일 제외)
- **픽셀 광고**: 페이지 전체 (보이지 않음)

### 12.3 수익 최적화
- **A/B 테스트**: 광고 위치, 크기, 디자인 테스트
- **타겟팅**: 카테고리별, 페이지별 광고 배치
- **다양한 광고 네트워크**: Google AdSense, 직접 광고주 등
- **프리미엄 광고**: 고수익 광고 우선 배치
