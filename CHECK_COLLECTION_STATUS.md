# 📊 데이터 수집 상태 확인 가이드

## 🔍 확인 방법

### 방법 1: GitHub Actions 실행 기록 확인 (가장 확실)

#### 1단계: Actions 페이지 접속
- URL: https://github.com/VENCEO86/realxbest/actions

#### 2단계: 워크플로우 확인
- **Daily Channel Collection** 워크플로우 클릭

#### 3단계: 실행 기록 확인
최근 3-4일간의 실행 기록을 확인하세요:

- ✅ **성공**: 초록색 체크 표시
  - 로그에서 "✅ 수집 완료" 메시지 확인
  - 데이터베이스에 데이터가 저장되었습니다

- ❌ **실패**: 빨간색 X 표시
  - 로그에서 오류 메시지 확인
  - 문제 해결 필요

- ⏳ **실행 중**: 노란색 원 표시
  - 현재 실행 중입니다

#### 4단계: 로그 확인
실행 항목을 클릭하여 상세 로그 확인:
- "🚀 데일리 자동 채널 수집 시작..."
- "✅ 데이터베이스 연결 성공"
- "📊 수집 통계: ..."
- "✅ 수집 완료"

---

### 방법 2: 웹사이트에서 실제 데이터 확인

#### 1단계: 메인 페이지 접속
- URL: https://realxbest.com

#### 2단계: 데이터 확인
- ✅ 랭킹 테이블에 채널 데이터가 표시되는지 확인
- ✅ 채널 이름, 구독자 수, 조회수 등이 표시되는지 확인
- ✅ 채널 상세 페이지 클릭하여 작동 확인

#### 3단계: 채널 상세 페이지 확인
- 채널 이름 클릭
- 성장 추이 차트 확인
- 데이터가 정상적으로 표시되는지 확인

---

### 방법 3: API 엔드포인트로 데이터 확인

#### 1단계: API 엔드포인트 접속
- URL: https://realxbest.com/api/rankings

#### 2단계: JSON 응답 확인
브라우저에서 JSON 데이터 확인:
```json
{
  "channels": [
    {
      "id": "...",
      "name": "...",
      "subscriberCount": 1234567,
      ...
    }
  ]
}
```

- ✅ 데이터가 있으면: 수집 성공
- ❌ 빈 배열이거나 오류: 수집 실패 또는 데이터 없음

- 어제 날짜
- 2일 전 날짜
- 3일 전 날짜

## ⚠️ 문제 해결

### GitHub Actions가 실행되지 않은 경우

1. **Secrets 확인**
   - https://github.com/VENCEO86/realxbest/settings/secrets/actions
   - `DATABASE_URL` ✅
   - `YOUTUBE_API_KEYS` ✅

2. **워크플로우 파일 확인**
   - `.github/workflows/daily-collect.yml` 파일 존재 확인
   - 스케줄 설정 확인: `cron: '0 18 * * *'`

3. **수동 실행 테스트**
   - Actions 페이지에서 "Run workflow" 버튼 클릭
   - 실행 로그 확인

### 데이터가 수집되지 않은 경우

1. **로그 확인**
   - GitHub Actions 실행 로그 확인
   - 오류 메시지 확인

2. **일반적인 문제**
   - API 키 할당량 초과
   - 데이터베이스 연결 실패
   - 스크립트 오류

3. **재실행**
   - "Run workflow" 버튼으로 수동 실행
   - 로그 확인하여 문제 해결
