이중 인코딩 문제:
  - 원인: UTF-8 데이터가 latin1 연결로 저장되면서
          각 바이트가 latin1 문자로 해석됨
  - 예시: "관리자" (UTF-8) → latin1로 저장 → 
          "ê´€ë¦¬ìžo" (이중 인코딩)
  - 해결: utf8mb4 연결로 올바른 UTF-8 데이터 재저장

SET NAMES utf8mb4의 효과:
  1. character_set_client = utf8mb4
     → 클라이언트가 보내는 쿼리를 utf8mb4로 해석
  2. character_set_connection = utf8mb4
     → 쿼리 처리 시 utf8mb4 사용
  3. character_set_results = utf8mb4
     → 결과를 utf8mb4로 반환

mb_internal_encoding('UTF-8'):
  → PHP의 내부 문자열 처리를 UTF-8로 설정
  → mb_* 함수들이 UTF-8로 동작

mb_http_output('UTF-8'):
  → HTTP 출력을 UTF-8로 설정
  → 출력 버퍼링 시 UTF-8로 변환
